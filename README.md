## Локальный запуск

Если запускать `python -m http.server` из корня репозитория, откроется корень проекта.

Используйте один из вариантов:

1. `npx quartz build --serve` — рекомендуемый способ для Quartz.
2. `npx quartz build` и затем сервер из папки `public`:
   - `cd public`
   - `python -m http.server 8000`

В корне проекта добавлен `index.html`-редирект на `./public/`, чтобы при случайном запуске сервера из корня сразу открывался собранный сайт.

## Правило для изображений в инфобоксе

Чтобы изображения в `ArticleInfobox` работали предсказуемо во всех статьях:

1. Кладите файлы в `content/assets/`.
2. В frontmatter указывайте `image`/`cover` с расширением файла (например, `Бакс.jpg`, `01-kostry-salema.png`).
3. Допускаются также явные пути (`./...`, `../...`, `/...`) и внешние `https://...` URL.

Значения без расширения (например, просто `01 Костры Салема`) считаются неоднозначными и не рендерятся как изображение в инфобоксе.
